The Klef Design System V2 is a **variable-driven CSS architecture** that uses CSS custom properties for declarative styling, inspired by Apple's design language.

## Philosophy

<Info>
  **"Classes define intention. Variables define behavior."**
  
  The design system separates semantic meaning (classes) from styling behavior (CSS variables), enabling highly composable and maintainable interfaces.
</Info>

## Variable Engine

The design system's most powerful feature is the **Variable Engine** - CSS that reads inline style variables:

<Tabs>
  <Tab title="Concept">
    Instead of writing utility classes like `.flex .items-center .gap-4`, you write inline CSS variables:
    
    ```html
    <!-- Traditional approach -->
    <div class="flex items-center justify-between p-5 bg-gray-100 rounded-lg">
    
    <!-- Variable-driven approach -->
    <div style="--flex; --items:center; --justify:space-between; --p:var(--sp-5); --bg:var(--bg-secondary); --br:var(--radius-lg)">
    ```
  </Tab>
  
  <Tab title="Implementation">
    The CSS uses attribute selectors to detect variables and apply styles:
    
    ```css
    /* Flexbox */
    [style*="--flex"] {
      display: flex;
    }
    
    [style*="--items:"] {
      align-items: var(--items);
    }
    
    [style*="--justify:"] {
      justify-content: var(--justify);
    }
    
    /* Spacing */
    [style*="--p:"] {
      padding: var(--p);
    }
    
    [style*="--gap:"] {
      gap: var(--gap);
    }
    ```
  </Tab>
  
  <Tab title="Benefits">
    <CardGroup cols={2}>
      <Card title="Type Safety" icon="check">
        Values come from design tokens, ensuring consistency
      </Card>
      <Card title="No Class Bloat" icon="broom">
        HTML stays clean and semantic
      </Card>
      <Card title="Composability" icon="layer-group">
        Variables combine freely without conflicts
      </Card>
      <Card title="Inspectability" icon="magnifying-glass">
        DevTools show actual CSS values
      </Card>
    </CardGroup>
  </Tab>
</Tabs>

## Available Variable Categories

The Variable Engine supports comprehensive styling:

### Layout & Positioning

```html
<div style="--pos:absolute; --top:0; --left:0; --z:var(--z-modal)">
<div style="--d:grid; --cols:repeat(3, 1fr); --gap:var(--sp-5)">
```

### Sizing

```html
<div style="--w:100%; --h:400px; --max-w:1200px">
<div style="--w-fill; --h-fill">  <!-- Fills available space -->
```

### Spacing

```html
<div style="--p:var(--sp-5); --mx:auto">
<div style="--py:var(--sp-8); --px:var(--sp-4)">
```

### Colors & Effects

```html
<div style="--bg:var(--blue); --c:var(--white)">
<div style="--shadow:var(--shadow-lg); --blur:20px">
```

### Typography

```html
<h1 style="--fs:var(--fs-4xl); --fw:var(--fw-bold); --lh:var(--lh-tight)">
<p style="--ta:center; --ls:var(--ls-tight)">
```

## Component Classes

While the Variable Engine handles behavior, **semantic component classes** define intention:

### Buttons

```css
.btn {
  display: inline-flex;
  align-items: center;
  justify-content: center;
  gap: var(--sp-2);
  padding: 12px 22px;
  font-family: inherit;
  font-size: var(--fs-lg);
  font-weight: var(--fw-regular);
  line-height: 1.17648;
  letter-spacing: var(--ls-tighter);
  border: none;
  border-radius: var(--radius-md);
  cursor: pointer;
  transition: all var(--duration-base) var(--ease-out);
}

.btn-primary {
  background: var(--blue);
  color: var(--white);
}

.btn-primary:hover:not(:disabled) {
  background: var(--blue-hover);
  transform: translateY(-1px);
  box-shadow: var(--shadow-md);
}
```

## Theming Support

The design system is built for theming:

```css
/* Light mode (default) */
:root {
  --text-primary: var(--k-black);
  --bg-primary: var(--white);
}

/* Dark mode (optional) */
@media (prefers-color-scheme: dark) {
  :root {
    --text-primary: #f5f5f7;
    --text-secondary: #86868b;
    --bg-primary: #000000;
    --bg-secondary: #1d1d1f;
    --border-light: rgba(255, 255, 255, 0.1);
  }
}
```

## Accessibility Features

    ```css
    @media (prefers-reduced-motion: reduce) {
      *, *::before, *::after {
        animation-duration: 0.01ms !important;
        transition-duration: 0.01ms !important;
        scroll-behavior: auto !important;
      }
    }
    ```
  </Accordion>
  
  <Accordion title="High Contrast">

    ```css

        --border-light: rgba(0, 0, 0, 0.3);
        --border-medium: rgba(0, 0, 0, 0.5);

    ```
  </Accordion>
